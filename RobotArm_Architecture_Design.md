# 桌面级轻负载金属机械臂全栈设计说明书

**项目目标：**
* 负载：0.5 ~ 1kg
* 精度：< 1mm 
* 特性：带力矩反馈、丝滑阻抗控制、笛卡尔空间走直线、重力前馈补偿
* 核心架构：分布式架构 (上位机UI -> ROS2控制大脑 -> 关节底层驱动)
* 关节成本下沉路线：基于 ESP32 + SimpleFOC 自研控制硬件

---

## 1. 硬件架构与选型 (底层物理核心)

### 1.1 电机选型策略 (阶梯递减与标准法兰结合法则)
为极度缩减末端转动惯量导致的重力静态偏转负载，6轴电机的体积和力量必须**严格从底座至末端呈现逐级变小**。考虑到机械件好加工与减速器好匹配，推荐采用“同法兰、不同机身长度”加末端微型化的方案：
1. **底座 / 大肩关节 (轴 1, 2)：** 
   - 承受全臂力矩，需极高抗扭要求。 
   - **选型推荐**：**4260 / 4240 标准法兰无刷电机**（最长身管，扭矩最爆） 或者 **4310 盘式电机**。（完美物理匹配 14 型号二手谐波减速器，由于法兰标准，螺丝孔位不用非标定制。推荐减速比 1:100）
2. **小臂 / 肘关节 (轴 3)：** 
   - 平衡发力的重点，承上启下。 
   - **选型推荐**：**4220 超薄型无刷 / 3508 盘式电机**。（搭配 14 型号或同口径轻量行星减速器）
3. **腕部与末端夹爪指姿态 (轴 4, 5, 6)：**
   - 负积极轻微，极致小巧减重！(坚决舍弃 42 尺寸，避免头重脚轻)
   - **选型推荐**：**2804 盘式 / 28mm口径微型无刷 (NEMA 11 尺寸)**。（搭配微型全金属行星减速器或者做成直驱丝杠）。

### 1.2 主控计算大脑 (Layer 2 运算中抠枢)
* **硬件设备**：树莓派 4B/5 或 香橙派 (Orange Pi 5)
* **系统环境**：安装 Ubuntu 22.04 LTS (或更高) 搭载 ROS2 框架底座 + MoveIt 2 组件。
* **唯一功能**：解算轨迹插补 / 各类型防撞运动学逆解 / 将分解切成极细碎的 100~500Hz 空间微刻度 / 透过其背面的外延 CAN HAT (插板) 下发给所有的机械手 ESP 底层。

### 1.3 关节驱动极客自研板物料榜 (Layer 3 - 每轴各一块)
* **MCU 平台核心双路线推荐：**
  - **选项 A (极致通讯与算力派)：`ESP32-S3`** 
    (双核 240MHz，包含 FPU。内置原生被隔离的 WiFi/CAN 协议栈，轻松分担 ROS 通信。缺点是自带 ADC 噪音大、非线性，需软件滤波补偿妥协)。
  - **选项 B (工业级丝滑控制派)：`STM32G431` (强烈推荐用于大扭矩关节)**
    (ST 家族专为高级电机 FOC 量身定制的核心！内置多个极其变态的极高精度与高线性度 ADC，以及自带直接绑定电机硬件定时器的注入同步触发功能，ADC 读数极其纯净，也是高级伺服电机的业界标杆)。
* **高端一体化电流采集与门驱芯片方案 (取代昂贵的外置组合)：**
  - **大关节方案 A (DRV8305 智能半桥 + 外部 MOS)**：DRV8305 内置 3路电流放大器，配合 6 颗 DFN3333 贴片 N 沟道 MOS管。省去高昂的外置 INA240 芯片，可应对极大扭矩爆发需求。
  - **大关节方案 B (DRV8316 极致全集成，强烈推荐)**：内部同时集成 8A 峰值 MOS管 与 电流采样放大器，无需外接任何功率管与采用电阻，极致压缩 PCB 面积，轻松驱动 4310 级别马达。
  - **末端微关节方案**：对于指尖或腕部极微电机，可直接使用更廉价小巧的 `DRV8313` (无电流采样需求时，靠跑 Voltage 模式 FOC 即可)。
* **高精度反馈感知：** 
  - 内闭环狂扫用：`MT6816 / AS5047P` (挂载在高速防断线 SPI 上！)。
  - 找寻机器静态死角（上电仅看一次）用：挂配防打滑物理轴最外侧极其便宜廉价的 `AS5600 (I2C)`。
* **高速无损差分防震防磁血管总线**：`SN65HVD230 (小型 CAN 收发器)`

## 2. 机械设计与 PCB 铺板布线规避

### 2.1 机械骨头铺设提示
* **抵御弯曲 (防抖绝招)**：
  - 臂体抛弃 3D 硬塑！使用极轻但是极硬的材料！比如找万能某宝定制二维：**厚碳纤维板**或者直接让数控简单进行**薄铝合金雕刻后用长柱搭建管状骨架！**（重量轻刚性高，且天生是个极好的“法拉第金属防爆笼”屏蔽网）。
* **杜绝弹性**：尽全部财力去买旧的二手精密**谐波减速器**装给大臂 1 和 2 轴。没有回程框量的物理降维打压才能让 FOC 的算盘完美落位。

### 2.2 PCB Layout 设计生死防雷坑：
* **必须隔离共模（极其关键）**：ESP32 的内部处理非常怕地线脉冲！`功率回路 GND (过电机MOS 的)` 和 `ESP数字弱电流 GND` 在铺铜时必须坚决**星型单点交汇连接！** 不可首端尾段随意交缠。
* **物理法拉第隔离与外天线转移**：由于你依然可能有调参时会需要动用到它的 WiFi 射频通讯需求，而 PCB 周围充满最要命的高高次交流高频斩波辐射干扰。**买带 U.FL 外接按扣端子的 ESP 模组（非板载梯形条天线型号）**，使用延长引针天线挂至机械骨外的塑胶孔透出！
* 电源进入处必须有 `大解耦铝电容 + 并联多个高频陶瓷旁路 (0.1uf 等)` 来为开关时补电蓄水吸噪。

## 3. 固件架构说明与功能开发实现法 (The Software Soul)

### 3.1 关节固件设计 (基于 SimpleFOC + C++ 原生化)
**坚决不放计算，只做打手闭环。**
* ESP32 程序主循环极简，将内反馈环 `Velocity`（防速超标）加上 `Torque` (闭环 FOC 力矩电流采集感知)，以最硬直的心态疯狂内卷。
* **开辟独立 RTOS 监听线程**：死守 ESP 内置 TWAI (CAN 收发口)。收到了树莓头盔在 100Hz 发过来的：`{ID: 1号，当期小刻度位置: 12.04度, 前馈预给向心力预期Torque: 0.1Nm, 预给前冲目标预置Velocity: +12r/s}` 马上往 FOC 层抛去强更追目标步！此手段即可完美压榨掉低级延迟时间差！

### 3.2 大制脑与动作插补（如何干各种极度高级的花活：ROS 2）

* **“走直线如何切法”**：
  电脑端 UI 或者上游控制下死命令“去到三维空间点 [X，Y，Z]”。树莓派内部由 `MoveIt2` 组件极速启用逆向矩阵接手解密，算准了整个线内沿途可能会让每个电机要怎么各司其职分配不同角度的路径曲线数组包 (极美的梯形轨迹圆滑切分后的密集数字流包)，打包并顺畅发给 CAN 线底下的大傻手下。

* **“机械臂一碰极速柔顺停摆退防怎么实现？”：**
  底层板的 `INA240` 从来没有休息过。它把探测到真实实际产生的力（反馈量），透过 CAN 总线实况告诉上了大主管。如果树莓大脑此刻发觉：哎不对，本来这条位置根本没有规划过阻挡物，但返回计算后发现 3号电机传回了剧烈爆高的 50% 额外实际力反抗！当即代码回调进入安全紧急中止 / 反向撤退安全缓冲模式。

* **如何拥有“反重力控制 (漂浮状态)”：**
  当底层 SimpleFOC 发力设定成极浅电流的时候，在树莓派中打开 **Pinocchio** 等高速数字矩阵引擎库，里面只需简单填好你这机械长短连杠自重，他就会算出：哪怕这刻不走，在如今这刁钻体态下，由于引力缘故，底下的大肩膀起码被白施加了 $Tau\_gravity$ 的力。于是树莓派无声无刻就在原本的空白通讯夹帧上面叠加上给这第一个马达预下发一个刚刚好抵御它重力的前置发力 FOC 。这时候，机械臂宛如“宇宙中无重力的手办失重展示件”——用手随便轻松顺推即滑动悬停！

### 4. 上位电脑的终局职现 (Layer 1 Pure Terminal)
它不需要懂得如何算矩阵，甚至不需要去管到底有多费脑子。你只是在 里面用 Python 挂上了 OpenCV。当你看到屏幕有个摄像头扫到了一杯水。Python 调用库识别杯子在这个画面坐标并发出抓取口令发包向系统树莓。剩下的脏活累活全被底层消化屏蔽隐离。
